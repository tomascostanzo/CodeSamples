Voici l'extrait d'un algorithme pour résoudre un problème combinatoire avec une approche las Vegas.
Trois types de noeuds: entree, étapes, Vue(racine).
À chaque sentier possible reliant deux points d’intérêt est associé un coût.
Resultat : Tracé des chemins qui répond aux contraintes et qui a un coût total minimal.
Dans le dossier DATA il y a les donnes qu'on peux executer pour tester l'algo.

Il s'execute sur eclipse. 
Premier argument : Le path vers le fichier de données. 
Deuxième argument : -t pour avoir le temps d'execution.
